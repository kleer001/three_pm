# three_pm ‚Äî "3pm"

A vanilla browser **roguelite**: five 16-year-olds at Merriton High try to get
home alive through a terrifying post-apocalypse cartoon-hellscape suburb. Each
run is one "day" ‚Äî a forced top-down **descent** south, with the camera window
auto-scrolling down and crushing stragglers. Home is the south edge; reach it to
win. Plain HTML, CSS, and ES modules ‚Äî no build system, bundler, or dependencies.

## Two layers: design corpus vs. the slice

- **`docs/00-index.md` ‚Ä¶ `docs/17-*.md` + `ENGAGEMENT.md`** are the design source
  of truth (engine, world, entities, combat, the five heroes, enemies, powerups,
  meta-progression, presentation, tone, scene flow, powerup synergies). Start at
  `docs/00-index.md`.
- **`src/`** is a **vertical slice** that proves the core loop. It deliberately
  implements a subset and diverges in places (e.g. combat is a freeze-slingshot
  stand-in, not the full per-hero kits). When code and spec disagree, the spec is
  the intent and the slice is the current reality ‚Äî reconcile rather than assume.

## Run, verify, ship

- **Run locally:** `./run.sh [port]` serves the folder and opens the game
  (`/`) plus the recorder (`/tools/record.html`). Or `python3 -m http.server 8000`.
  ES modules require HTTP, not `file://`.
- **Verify logic headlessly:** `node tests/smoke.mjs` ‚Äî exercises the pure,
  browser-free logic (generation, connectivity, collision, BFS pathfinding,
  wall-density knob). This environment has **no headless browser**, so visual /
  input behavior must be playtested in a real browser; say so rather than
  claiming an unverified visual fix works.
- **Package for itch.io:** `./package.sh` ‚Üí `dist/three_pm.zip` (index.html at
  the zip root; `tools/` and `docs/` excluded).
- **GitHub Pages:** auto-served from `main` at <https://kleer001.github.io/three_pm/>.
- **Deploy invariant:** `index.html` stays at the repo root and **all paths are
  relative** (no leading `/`) ‚Äî a leading slash 404s on Pages (served under
  `/three_pm/`) and breaks inside itch.io's iframe.

## Project structure

- `index.html` ‚Äî page shell; loads `src/main.js` as a module
- `src/main.js` ‚Äî boot: canvas, input, loop; runs each day as two phases
  (weapon-select ‚Üí run ‚Üí select for the next day)
- `src/core/loop.js` ‚Äî fixed-timestep loop (owns no game state)
- `src/core/rng.js` ‚Äî seeded PRNG (mulberry32) + labelled sub-streams
- `src/run/levelgen.js` ‚Äî suburb generator: street grid ‚Üí houses ‚Üí noise-decay
  obstacles ‚Üí connectivity repair. Exposes `wallScaleX/Y` + `wallDensity` knobs
- `src/run/collision.js` ‚Äî center-based AABB vs walkable tiles
- `src/run/selectScene.js` ‚Äî per-run weapon picker shown before each descent
- `src/run/runScene.js` ‚Äî the playable RUN scene (descent, chosen weapon, enemies,
  bodies, render). Hosts the `BEHAVIORS` registry (spec 06's `chaser`/`swarmer`/
  `shooter`/`charger`). Gameplay knobs live in `balance.js` (`HERO`, `weapons`,
  `ENEMIES` roster, `director`, `SCROLL`, `MAP_H`, `FREEZE_DUR`, wall scale/density)
- `src/run/combat.js` ‚Äî the one resolver (spec 03/04): `recomputeDerived` (1‚Äì10
  `stats`‚Üí`derived`: moveSpeed/maxHp/dmgResist/knockback/maxMana/abilityPower),
  percent-HP + stat-scaled `weaponDamage`, `applyDamage` (i-frames/dmgResist/death),
  mana (`regenMana`/`canCast`/`spendMana`) ‚Äî shared by hero and enemies
- `src/run/director.js` ‚Äî spec 06 spawn director: distance-scaled threat budget +
  `distanceBand` eligibility, placing enemies off-screen home-ward
- `src/ai/ai.js` ‚Äî BFS pathfinding + waypoint pickers (ported from BrainMaze)
- `src/input/input.js` ‚Äî movement on WASD, arrows, and vim `hjkl`; `SPACE` fires
- `tests/smoke.mjs` ‚Äî node logic tests
- `tools/` ‚Äî `record.html` (local screen recorder) and the art-generation scripts
- `art-test/` ‚Äî art preview sandbox
- `assets/` ‚Äî generated sprite/tile atlases (art)
- `package.sh` ¬∑ `run.sh` ¬∑ `.nojekyll` ¬∑ `.scaffold.json`

## Art is isolated from game code

Art (`assets/*.png/json`, generated by `tools/gen_*.py`, previewed in `art-test/`)
is **droppable**: the game renders procedurally and must boot without it. Do not
edit `src/` for art reasons, and do not let the game hard-depend on a sprite
asset existing in a way that white-screens on load.

## Code style

- `camelCase` for functions/vars, `PascalCase` for classes (standard JS).
- ES modules with explicit `import`/`export`. No globals.
- Comments explain *why*, not *what*.

## Git
